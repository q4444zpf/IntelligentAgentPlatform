# 通用智能体平台功能设计

## 1. 设计目标

本功能设计面向通用智能体平台第一版建设，重点完成可配置、可编排、可协同、可审计的智能体平台底座。

第一版必须支持：

- 智能体创建、配置、发布和调用。
- Prompt 模板、变量、版本和调试。
- Skill / Tool 注册、参数配置、调试和调用留痕。
- 用户维护自己的智能体、MCP、Skill，并可提交发布到公用资源库。
- Web 版智能体采用沙箱执行模式，防止用户自建能力影响平台服务器安全。
- 知识库上传、解析、检索和问答。
- 拖拽式流程编排基础版。
- 编排式多智能体协同。
- 多轮 AI 对话和会话管理。
- 大模型配置和调用记录。
- 用户、角色、权限和基础审计。
- OpenAPI 基础调用能力。

第一版暂不追求：

- 完整 BPM 流程引擎。
- 大规模自治型多智能体。
- 商业化插件市场。
- 复杂多租户计费。
- 桌面端深度本地操作。
- 行业专用模型能力。

| 角色 | 主要职责 |
|---|---|

### 3.1 资源空间设计

平台资源分为个人资源和公用资源。

| 资源空间 | 说明 |
|---|---|
| 个人资源 | 用户自己创建和维护的智能体、MCP、Skill、Prompt、知识库和流程，默认仅本人可见 |
| 团队资源 | 部门或项目组内共享的资源，可按角色或成员授权 |
| 公用资源 | 平台内所有授权用户可查看和使用的资源，需要通过发布审核 |
| 系统内置资源 | 平台预置资源，由系统管理员维护 |

### 3.2 公用发布机制

用户创建的智能体、MCP 和 Skill 默认属于个人资源。用户可以将资源提交发布到公用资源库，审核通过后其他用户可以搜索、查看、复制或使用。

发布流程：

```text
用户创建个人资源
→ 完成配置和调试
→ 提交发布到公用
→ 系统进行基础校验
→ 审核人员审核名称、描述、权限、安全风险和可用性
→ 审核通过后进入公用资源库
→ 其他用户可使用或复制
→ 原作者可提交新版本或申请下架
```

发布状态：

| 状态 | 说明 |
|---|---|
| 私有草稿 | 仅创建人可见，可编辑 |
| 私有可用 | 创建人可使用，也可授权给指定成员 |
| 待审核 | 已提交公用发布，等待审核 |
| 审核驳回 | 未通过审核，需要修改后重新提交 |
| 公用已发布 | 已进入公用资源库 |
| 公用已下架 | 曾经发布过，当前不可被新用户使用 |

## 8. Skill / Tool 管理

### 8.1 功能说明

Skill / Tool 管理用于把外部 API、内部服务、数据查询、文件处理、系统动作等能力注册为智能体可调用工具。

### 8.2 Skill 类型

- HTTP API
- MCP Server
- 内置函数
- 脚本任务
- 数据库查询
- 文件处理
- 消息通知
- 人工确认
- 业务系统动作

### 8.3 Skill 字段

| 字段 | 说明 |
|---|---|
| Skill 名称 | 展示名称 |
| Skill 编码 | 唯一标识 |
| Skill 类型 | HTTP API、脚本、内置函数等 |
| 描述 | 给用户和大模型看的能力说明 |
| 调用地址 | API 地址或执行入口 |
| 请求方法 | GET、POST 等 |
| 输入 Schema | JSON Schema |
| 输出 Schema | JSON Schema |
| 鉴权方式 | 无、Token、密钥、OAuth 等 |
| 超时时间 | 单次调用超时 |
| 重试次数 | 失败重试策略 |
| 是否需确认 | 调用前是否需要人工确认 |
| 所属空间 | 个人、团队、公用、系统内置 |
| 创建人 | 资源所有者 |
| 权限范围 | 可使用用户、角色或智能体 |
| 发布状态 | 私有草稿、私有可用、待审核、公用已发布、已下架 |
| 运行状态 | 启用、停用 |

### 8.4 Skill 调试流程

```text
填写 Skill 基础信息
→ 配置输入输出 Schema
→ 配置调用地址和鉴权
→ 输入测试参数
→ 发起测试调用
→ 查看请求、响应、耗时和错误
→ 保存为个人可用
→ 可选择提交发布到公用
→ 绑定到智能体或流程节点
```

### 8.5 安全控制

- 参数 Schema 强校验
- 敏感字段脱敏展示
- 高风险 Skill 调用前人工确认
- 调用权限校验
- 超时控制
- 调用日志留痕
- 失败原因记录
- Web 版智能体调用用户自建 Skill 时必须通过沙箱或受限代理执行
- 禁止用户自建 Skill 直接访问服务器宿主文件系统、系统命令和平台密钥
- 第三方 API 类 Skill 默认禁止访问内网敏感地址，需通过白名单放行

### 8.6 Skill 公用发布

用户可以维护自己的 Skill，并将稳定可复用的 Skill 提交到公用资源库。

提交公用发布时需要校验：

- Skill 名称、描述和使用说明完整。
- 输入输出 Schema 已配置。
- 调试测试至少成功一次。
- 鉴权信息不包含个人明文密钥。
- 高风险动作已配置人工确认。
- 权限范围和调用限额已配置。
- 网络访问范围已配置，内网地址访问需管理员审核。
- 需要服务器侧执行的任务已配置沙箱执行。

审核通过后，公用 Skill 可以被其他用户绑定到自己的智能体或流程中。其他用户默认只能使用，不允许直接修改原始 Skill；如需改造，应复制为个人 Skill。

## 8A. MCP 管理

### 8A.1 功能说明

MCP 管理用于维护可被智能体调用的 MCP Server。用户可以创建和维护自己的 MCP 配置，也可以将通用 MCP 发布到公用资源库。

### 8A.2 MCP 类型

- 本地 MCP Server
- 远程 MCP Server
- 平台托管 MCP Server
- 第三方 MCP Server

### 8A.3 MCP 字段

| 字段 | 说明 |
|---|---|
| MCP 名称 | 展示名称 |
| MCP 编码 | 唯一标识 |
| MCP 类型 | 本地、远程、平台托管、第三方 |
| 描述 | 能力说明 |
| 连接方式 | stdio、HTTP、SSE 等 |
| 启动命令 | 本地或托管 MCP 的启动命令 |
| 服务地址 | 远程 MCP 地址 |
| 环境变量 | 启动或连接所需配置，敏感值加密存储 |
| 可用工具 | MCP 暴露的 tools 列表 |
| 可用资源 | MCP 暴露的 resources 列表 |
| 健康状态 | 可用、异常、未检测 |
| 所属空间 | 个人、团队、公用、系统内置 |
| 创建人 | 资源所有者 |
| 权限范围 | 可使用用户、角色或智能体 |
| 发布状态 | 私有草稿、私有可用、待审核、公用已发布、已下架 |
| 运行状态 | 启用、停用 |

### 8A.4 MCP 管理功能

- 新建 MCP
- 编辑 MCP
- 测试连接
- 同步工具列表
- 查看工具 Schema
- 绑定到智能体
- 绑定到流程节点
- 启用 / 停用
- 提交发布到公用
- 撤回发布申请
- 从公用资源复制为个人 MCP
- 查看调用日志

### 8A.5 MCP 发布校验

提交公用发布时需要校验：

- MCP 连接测试成功。
- 工具列表可正常同步。
- 工具名称、描述和参数 Schema 完整。
- 环境变量和密钥未明文暴露。
- 文件、网络、系统命令等高风险能力已标记。
- 高风险工具已配置人工确认或权限限制。
- 本地或平台托管 MCP 已配置沙箱运行。
- 远程 MCP 已配置网络访问白名单或风险说明。

### 8A.6 MCP 调用流程

```text
智能体选择 MCP 工具
→ 校验用户、智能体和 MCP 权限
→ 注入必要上下文和参数
→ 通过沙箱或受限代理调用 MCP Server
→ 获取工具结果
→ 记录 MCP、工具、入参、出参、耗时和错误
→ 将结果返回给智能体或流程节点
```

### 8A.7 Web 智能体沙箱要求

Web 版智能体使用 MCP 时必须遵守沙箱要求：

- 普通用户配置的本地 MCP 不允许直接运行在平台 API 服务进程内。
- 平台托管 MCP 必须运行在隔离容器或受限 Worker 中。
- 每次执行分配独立临时目录，执行结束清理。
- 默认禁止访问宿主机敏感目录。
- 默认禁止访问内网敏感地址。
- 限制执行时长、CPU、内存和并发。
- MCP 工具调用必须记录执行人、智能体、MCP、工具、参数摘要、耗时和结果状态。

| 成功 | 流程正常结束 |
| 失败 | 节点或流程异常 |
| 已取消 | 用户取消执行 |

### 10.7 流程运行链路

```text
用户手动触发或 API 触发
→ 创建流程实例
→ 读取发布版本
→ 执行开始节点
→ 按连线顺序执行节点
→ 每个节点记录输入、输出、状态和耗时
→ 遇到人工确认节点暂停
→ 人工处理后继续执行
→ 生成流程结果
→ 记录完整运行日志
```

## 11. 多智能体协同

### 11.1 功能说明

多智能体协同是第一版必做核心功能。第一版采用编排式协同，不做完全开放的自治协同。

### 11.2 协同方式

- 主从协同：主智能体拆解任务，子智能体执行。
- 流水线协同：多个智能体按顺序处理。
- 汇总协同：多个智能体输出结果后，由汇总智能体生成最终结论。
- 人机协同：关键节点需要用户确认。

### 11.3 协同配置字段

| 字段 | 说明 |
|---|---|
| 协同名称 | 展示名称 |
| 主智能体 | 负责任务规划或汇总 |
| 子智能体 | 参与协同的智能体列表 |
| 执行顺序 | 顺序执行或指定编排顺序 |
| 输入映射 | 上下文如何传给各智能体 |
| 输出映射 | 各智能体结果如何汇总 |
| 人工确认 | 是否需要确认 |
| 超时时间 | 协同任务最长执行时间 |

### 11.4 协同运行流程

```text
用户发起协同任务
→ 加载协同配置
→ 主智能体生成任务说明
→ 子智能体按配置依次执行
→ 每个子智能体输出结构化结果
→ 汇总智能体生成最终回复
→ 必要时进入人工确认
→ 记录协同任务全过程
```

### 11.5 协同日志

需要记录：

- 协同任务 ID
- 发起人
- 主智能体
- 子智能体列表
- 每个智能体输入
- 每个智能体输出
- 模型调用记录
- Tool 调用记录
- 耗时
- 状态
- 失败原因

## 17. 第一版验收标准

### 17.1 智能体

- 可以创建并发布智能体。
- 可以绑定大模型、Prompt、知识库和 Skill。
- 普通用户可以维护自己的私有智能体。
- 普通用户可以将个人智能体提交发布到公用资源库。
- 审核通过后，其他用户可以使用或复制公用智能体。
- 普通用户可以选择已授权智能体进行对话。
- 智能体调用过程有日志。
- Web 版智能体调用 Tool / MCP 时经过沙箱或受限代理执行。

### 17.2 Prompt

- 可以创建、编辑、发布 Prompt。
- 可以配置变量并进行调试。
- 智能体可以引用已发布 Prompt。

### 17.3 Skill / Tool

- 可以注册 HTTP API 类型 Skill。
- 可以配置输入输出 Schema。
- 可以调试调用。
- 普通用户可以维护自己的私有 Skill。
- 普通用户可以将个人 Skill 提交发布到公用资源库。
- 审核通过后，其他用户可以使用或复制公用 Skill。
- 可以绑定到智能体和流程节点。
- 调用记录可查询。
- 用户自建 Skill 不允许直接在平台 API 服务进程内执行不受控代码。

### 17.4 MCP

- 可以创建个人 MCP 配置。
- 可以测试 MCP 连接。
- 可以同步 MCP 工具列表。
- 可以将 MCP 绑定到智能体。
- 普通用户可以将个人 MCP 提交发布到公用资源库。
- 审核通过后，其他用户可以使用或复制公用 MCP。
- MCP 调用记录可查询。
- 本地或平台托管 MCP 必须在沙箱或受限 Worker 中运行。

### 17.5 知识库

- 可以创建知识库。
- 可以上传并解析文档。
- 可以完成向量化和检索。
- AI 回答可以展示引用来源。

### 17.6 流程编排

- 可以拖拽创建基础流程。
- 可以配置智能体节点、Skill 节点、条件节点和人工确认节点。
- 可以发布流程。
- 可以手动或 API 触发流程。
- 可以查看流程实例和节点运行记录。

### 17.7 多智能体协同

- 可以配置一个主智能体和多个子智能体。
- 可以按顺序执行子智能体。
- 可以汇总多个智能体输出。
- 可以查看每个智能体的输入、输出和调用日志。

### 17.8 权限、发布与审计

- 不同角色只能访问授权菜单和资源。
- 智能体、MCP、Skill、知识库、流程调用前需要权限校验。
- 个人资源默认仅创建人可见。
- 提交公用资源需要审核。
- 公用资源发布、驳回、下架过程可审计。
- Web 智能体的高风险执行不能直接访问服务器宿主环境。
- 关键操作和调用行为可审计。
