# 通用智能体平台建设方案

## 1. 平台定位

建设一个面向多行业、多系统、多终端的通用智能体平台，支撑智能体创建、提示词管理、Skill / Tool 管理、知识库问答、流程编排、多智能体协同、专有 AI 对话、多用户访问、大模型接入、第三方系统集成和 Web / 桌面 / 客户端嵌入。

平台第一阶段先聚焦通用智能体能力底座，不绑定具体行业业务，不预设特定领域模型。后续可在平台底座之上接入政务、企业运营、工程管理、客服、办公自动化、数据分析、知识问答、业务辅助决策等行业场景。

核心目标：

- 统一管理智能体、Prompt、Skill、Tool、知识库、流程和大模型配置。
- 支持 Web 端、桌面端、嵌入式 AI 对话框、OpenAPI 和 SDK 等多种访问方式。
- 支持智能体按运行端、用户角色、业务系统和上下文来源进行差异化配置。
- 支持确定性工具调用、RAG 检索增强、多轮对话、任务执行和人工确认。
- 支持拖拽式流程编排，实现数据、工具、智能体、条件判断和人工节点组合。
- 支持企业级权限、租户隔离、审计日志、调用统计、监控告警和安全治理。

## 3. 智能体管理

智能体是平台能力的核心组织单元。一个智能体由角色定位、提示词、可用模型、可用 Skill、知识库、运行端配置、权限策略和安全策略共同组成。

### 3.1 智能体类型

| 类型 | 说明 |
|---|---|
| 问答型智能体 | 面向知识库、文档、制度、FAQ 的问答 |
| 助手型智能体 | 面向办公、分析、写作、摘要、翻译等通用任务 |
| 工具型智能体 | 主要负责调用外部 API、系统动作和自动化工具 |
| 数据分析智能体 | 面向结构化数据查询、统计、图表和洞察生成 |
| 流程型智能体 | 作为流程节点参与业务编排和任务流转 |
| 审核型智能体 | 对内容、参数、结果、风险进行辅助审核 |
| 协同型智能体 | 与其他智能体协作完成复杂任务 |
| 自定义智能体 | 由平台用户按场景自行配置 |

### 3.2 智能体配置结构

```text
智能体
├─ 基础配置
│  ├─ 名称
│  ├─ 描述
│  ├─ 类型
│  ├─ 头像 / 图标
│  └─ 启用状态
├─ 能力配置
│  ├─ 系统提示词
│  ├─ 可用大模型
│  ├─ 可用知识库
│  ├─ 可用 Skill / Tool
│  └─ 输出格式要求
├─ 端侧配置
│  ├─ Web 端配置
│  ├─ 桌面端配置
│  ├─ 嵌入式对话框配置
│  └─ API 调用配置
├─ 权限配置
│  ├─ 可见范围
│  ├─ 可使用角色
│  ├─ 可调用工具
│  └─ 数据访问范围
└─ 治理配置
   ├─ 敏感词策略
   ├─ 人工确认策略
   ├─ 调用限额
   └─ 审计策略
```

### 3.3 Web 端与桌面端差异

| 配置项 | Web 端 | 桌面端 |
|---|---|---|
| 交互方式 | 浏览器页面、嵌入式对话框、业务系统侧边栏 | 桌面客户端、托盘助手、快捷键唤起、本地窗口 |
| 提示词 | 偏页面上下文、表单字段、业务对象、系统权限 | 偏本地文件、客户端状态、用户操作环境 |
| 上下文来源 | 当前页面、路由参数、接口数据、用户权限 | 本地文件、剪贴板、客户端缓存、设备状态 |
| 工具调用 | Web API、业务系统接口、知识库、在线工具 | 本地文件处理、桌面通知、本地应用联动、Web API |
| 权限控制 | 系统账号、角色、菜单、接口权限 | 客户端登录态、本地权限、设备绑定、接口权限 |
| 输出形式 | 页面回复、图表、表单填充、流程触发 | 弹窗回复、文件生成、任务提醒、本地应用联动 |

## 5. Skill / Tool 管理

Skill / Tool 是智能体可调用的确定性能力，用于把大模型从“生成回答”扩展到“查询、计算、处理、执行”。

### 5.1 Skill 分类

- 数据查询类：查询数据库、接口、表格、指标等结构化数据。
- 文件处理类：读取、解析、转换、生成 Word、Excel、PDF、图片等文件。
- 知识检索类：检索知识库、文档库、向量索引、FAQ。
- 内容生成类：生成报告、摘要、邮件、通知、图表说明。
- 系统动作类：创建工单、发送消息、回写系统、触发流程。
- 外部 API 类：封装第三方系统接口、SaaS API、企业内部服务。
- 代码执行类：执行受控脚本、数据分析代码、计算任务。
- 人工协同类：发起确认、审批、转人工、收集补充信息。

### 5.2 管理能力

- Skill 注册
- 参数 Schema 定义
- 返回值 Schema 定义
- 权限配置
- 调试测试
- 版本管理
- 启用 / 停用
- 绑定智能体
- 调用统计
- 异常日志
- 超时与重试配置
- 人工确认配置

### 5.3 Skill 注册字段

| 字段 | 说明 |
|---|---|
| Skill 名称 | 工具展示名称 |
| Skill 标识 | 系统内部唯一编码 |
| Skill 类型 | 数据查询、文件处理、外部 API、系统动作等 |
| 调用方式 | HTTP API、函数调用、脚本、消息队列、插件等 |
| 输入参数 | JSON Schema 或表单参数 |
| 输出结果 | JSON Schema、文本、文件、图表等 |
| 权限范围 | 用户、角色、部门、租户、智能体 |
| 超时设置 | 单次调用最长执行时间 |
| 重试策略 | 失败后是否重试及重试次数 |
| 审计策略 | 是否记录入参、出参、调用人和耗时 |

## 9. 多智能体协同

多智能体协同是平台必做核心能力，不作为后续可选增强项。第一版应优先实现可控、可审计、可复现的协同机制，避免过早引入不可解释的完全自治链路。

### 9.1 协同模式

- 主从模式：主智能体规划任务，专业智能体执行子任务。
- 会商模式：多个智能体分别给出分析意见，汇总智能体生成结论。
- 流水线模式：上一个智能体输出作为下一个智能体输入。
- 投票模式：多个智能体独立判断，系统聚合结果。
- 人机协同模式：智能体提出建议，用户审核、调整和确认。

### 9.2 协同治理

- 协同任务记录
- 子任务拆解记录
- 每个智能体输入输出留痕
- 工具调用留痕
- 冲突意见标记
- 汇总结论可追溯
- 人工确认节点
- 协同超时与终止策略

### 9.3 第一版必做边界

第一版多智能体协同先支持“编排式协同”，即通过流程或配置明确各智能体职责、执行顺序和汇总方式。

必做能力：

- 一个主智能体协调多个子智能体
- 子智能体按顺序执行
- 子智能体可绑定不同 Prompt、知识库和 Skill
- 上一个智能体输出可作为下一个智能体输入
- 汇总智能体生成最终结果
- 每个智能体的输入、输出、模型调用和 Tool 调用留痕
- 协同任务状态记录
- 协同失败时可查看失败智能体和失败原因
- 关键步骤支持人工确认

第一版暂缓能力：

- 智能体自主动态拉起任意智能体
- 大规模群体讨论
- 自动投票与复杂仲裁
- 长周期自治任务
- 跨租户智能体协同
- 完全开放的智能体插件生态

## 15. 通用能力场景

### 15.1 知识库问答

```text
用户提问
→ 权限校验
→ 检索知识库
→ 重排检索结果
→ 大模型生成回答
→ 返回引用来源
→ 记录会话与反馈
```

### 15.2 工具调用助手

```text
用户提出任务
→ 智能体识别意图
→ 选择可用 Skill
→ 参数补全与确认
→ 调用 Skill
→ 解释执行结果
→ 记录调用日志
```

### 15.3 流程自动化

```text
触发流程
→ 获取输入数据
→ 执行条件判断
→ 调用智能体或 Skill
→ 人工确认
→ 输出结果 / 回调系统
→ 归档运行记录
```

### 15.4 嵌入式 AI 对话框

```text
第三方系统打开页面
→ AI 对话框注入页面上下文
→ 用户发起提问
→ 智能体结合页面对象、知识库和工具回答
→ 必要时触发流程或回写系统
```
